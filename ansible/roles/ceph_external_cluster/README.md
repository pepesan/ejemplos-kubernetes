# ceph_external_cluster

Bootstraps a standalone Ceph cluster via `cephadm` on a set of already-provisioned instances — one
monitor node plus a set of OSD nodes — completely independent of any Kubernetes cluster. Faithfully
extracted from `ansible/base/05_k8s_ha_almacenamiento_persistente_externo_ceph`:

1. Installs `cephadm`/`ceph-common` and the `ceph` system user/group on every Ceph node.
2. Bootstraps the cluster with `cephadm bootstrap` on the monitor.
3. Distributes the cluster's own SSH key (generated by cephadm) to every OSD host.
4. Adds the OSD hosts to the orchestrator and provisions OSDs on every available raw disk.

Pair this with [`k8s_ceph_external_csi`](../k8s_ceph_external_csi/) to wire a separately-provisioned
Kubernetes cluster (e.g. via `k8s_ha_cluster` with `csi: none`) to consume the cluster this role builds.
The two are deliberately separate roles — this one has no idea a Kubernetes cluster even exists, matching
the "odd one out" nature of this backend among `k8s_ha_cluster`'s CSI options (see `roles/PLAN.md`).

## Requirements

- Ubuntu 24.04/26.04 or Debian 12 (bookworm)/13 (trixie) instances.
- Every OSD host needs a raw, unformatted block device beyond its root disk (Ceph scans and claims every
  available empty disk) — see `lxd_machine_provision`'s `lxd_extra_disk` host var, which this role's own
  Molecule scenario uses to provision one.
- A single invocation covering both node roles: run against `hosts: ceph_monitors:ceph_osds` (or
  equivalent) in one play — same "task files guard themselves by group membership" convention
  `k8s_ha_cluster` uses.

## Role Variables

| Variable | Default | Purpose |
| --- | --- | --- |
| `ceph_external_cluster_monitors_group` | `ceph_monitors` | Single-host group: the node that runs `cephadm bootstrap` |
| `ceph_external_cluster_osds_group` | `ceph_osds` | OSD nodes |
| `ceph_external_cluster_dashboard_password_path` | `{{ inventory_dir }}/ceph_dashboard_password.txt` | Where the generated Ceph Dashboard admin password is saved/read back from |

## Example Playbook

```yaml
- hosts: ceph_monitors:ceph_osds
  roles:
    - ceph_external_cluster
```

Pointing this role at `base/05`'s own inventory needs zero changes — the default group names match
exactly.

## Testing

```bash
cd ansible/roles/ceph_external_cluster
molecule test
```

1 monitor + 3 OSD VMs — 3 is a hard floor here, not a arbitrary choice: the role's own OSD-provisioning
wait can never succeed with fewer (mirrors `k8s_ha_cluster`'s Rook Ceph `mon.count: 3` floor). Uses the
`default` driver against the real host (this role provisions actual Ceph daemons via `cephadm`, not
something a Docker/Podman-based Molecule driver could do), with `lxd_machine_provision`'s `lxd_extra_disk`
providing each OSD's backing disk. Includes Molecule's native `idempotence` step.
